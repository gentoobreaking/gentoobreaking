Octopussy — 另一套 Syslog 與 Log 管理利器

突然看到很久之前看過的軟體，一直沒機會測試，這次仔細看看還真不錯！一定要找時間來玩玩 ^^

官方網站：https://www.octopussy.pm

GitHub：github.com/Octopussy-Project/Octopussy

Octopussy 功能總覽

功能類別 說明
LDAP 支援 支援 LDAP 認證，可與企業目錄整合管理使用者和聯絡人
多元警示通知 可透過 Email、IM（Jabber）、NSCA（Nagios）、Zabbix_sender 等管道發送告警
報表匯出 支援 Email、FTP、SCP 等方式匯出報表
架構圖（Maps） 可建立網路架構圖，視覺化呈現監控拓撲
Plugin 擴充架構 提供 Input / Output Plugin，可擴充報表輸入輸出格式
報表排程 可預設時間自動產生報表
RRDTool 圖形化 使用 RRDTool 將 Syslog 活動繪製成統計圖表
線上更新 支援 Services、Tables、Multilanguage 線上更新機制
多國語言 支援 English、French、German、Spanish、Portuguese、Russian、Italian、Turkish 等語系
主題化介面 介面與報表皆可更換主題
Wizard 精靈 提供 Wizard 精靈，輕鬆建立新訊息 / 服務以解析無法識別的 Log
被動式監控 接收 Syslog 訊息，無需在受控設備上安裝代理程式

支援的服務類型

💡 Octopussy 支援任何支援 Syslog 協定的設備（伺服器、路由器、交換器、防火牆、負載平衡器等），並內建豐富的服務解析器：

分類 支援項目
網路設備 Cisco Router & Switch (IOS/ASR), Juniper Netscreen FW/NSM, Fortinet FW, F5 BigIP, DenyAll Reverse Proxy, HP-Tools, Neoteris/Juniper FW
作業系統 / 系統 Linux (Kernel/System/PAM/Auditd/IPTables/AppArmor), BSD, Sshd, Syslog-ng, Monit, Xen, VMware ESX(i)
資料庫 MySQL, PostgreSQL
網路服務 Apache 2, Bind, Squid, Postfix, Snmpd, Drbd, NetApp NetCache, Samba, TACACS
監控 / 安全 Nagios, Samhain, ClamAV, Ironport MailServer, LDAP
Windows 支援 Windows Snare Agent, Windows System

Octopussy 與其他工具的定位差異

工具 主要定位 適合場景
Octopussy Syslog 被動監控 + 即時警示（SIEM 解決方案） 10-100 台伺服器環境，快速警示而非長期儲存
ELK / Graylog 日誌集中儲存與全文檢索 需要長期留存與關鍵字搜尋的環境
Cacti RRDTool 圖形化 SNMP 效能監控 流量、CPU、記憶體等效能指標
Nagios / OpenNMS 主機 / 服務可用性監控與告警 系統存活、服務狀態、臨界值告警
rsyslog + MySQL Syslog 收集與集中儲存 做為後端 log 集中管道，Octopussy 讀取分析

✅ Octopussy 是一套功能完整的被動式 Log 分析平台，採用 Perl 開發，無需在被控設備上安裝代理程式，適合管理多台網路設備與伺服器的 IT 人員。最新穩定版為 v1.0.16（2017-06-03），項目已遷移至 GitHub Pages 與 GitHub 託管。
