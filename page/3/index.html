網站分析軟體

發表於 2008-10-21 更新於 2026-06-18 分類於 開發工具與自動化

網頁流量統計工具比較：AWStats vs Webalizer

與 長官聊到網頁分析統計軟體，主要需求是統計user download file 的 bandwidth，希望能依目錄區分下載流量、列出被下載的檔案清單及流量報表。

1. AWStats

💡 Free real-time logfile analyzer to get advanced statistics

官方網站：http://awstats.sourceforge.net/

線上 Demo 展示

類型 連結
Web Server Log 分析 Demo - Web
FTP Log 分析 Demo - FTP
Mail Log 分析 Demo - Mail

功能特色

  • 支援 Web、FTP、Mail、Squid、Streaming 多種 Log 格式
  • 即時更新（可關閉即時功能僅保留靜態報表）
  • 可統計下載檔案、錯誤頁面、訪客来源、瀏覽器、作業系統等
  • 可依目錄或副檔名篩選，適合下載目錄流量分析需求
  • 輸出 HTML 報表，瀏覽器即可檢視

2. Webalizer

💡 The Webalizer is a fast, free web server log file analysis program. It produces highly detailed, easily configurable usage reports in HTML format, for viewing with a standard web browser.

官方網站：http://www.webalizer.org/

台灣 ADJ 網路 Demo：http://www.adj.idv.tw/usage/usage_200810.html

功能特色

  • 速度快、系統資源佔用低，適合大量 Log 檔案
  • 輸出 HTML 格式報表，含長條圖、線圖等視覺化圖表
  • 支援 ISP 匿名化（Hideous/hide IP）功能
  • 可依國家、地區、URL、Referer 等維度分析
  • Unix / Linux 平台為主，也有 Windows 版

AWStats vs Webalizer 比較

比較項目 AWStats Webalizer
授權方式 GPL 開源 GPL 開源
分析速度 中等 較快
支援 Log 類型 Web / FTP / Mail / Squid Web / FTP 為主
目錄層級分析 支援，可設定過濾規則 有限支援
下載檔案統計 ✅ 可統計下載次數與流量 可統計
GeoIP 國家分析 需搭配 GeoIP 插件 內建支援
圖形報表 靜態 PNG 圖表 長條圖、線圖、分布圖
設定彈性 高度彈性，Plugin 擴充 基本設定，够用就好

結論與建議

🎯 根據需求的選擇建議：

需求情境 推薦工具 原因
需要目錄層級的下載流量統計 AWStats 可設定目錄過濾規則，彈性較高
Log 檔案量很大，需要快速出報表 Webalizer 處理速度快，資源佔用低
同時要統計 FTP / Mail / Web 多種服務 AWStats 原生支援多類型 Log
不想安裝太多東西，快速部署 Webalizer 設定簡單，上手快

⚠️ 兩者皆為離線分析工具，需定期執行更新作業（建議加入 crontab 每小時或每日自動更新）。若需要即時查看流量，建議搭配前面提到的 Apache Log To Syslog 機制蒐集資料，再由 AWStats/Webalizer 統一報表輸出。

資安警示：即時通訊視訊監控工具的傳播模式與自保之道

發表於 2008-10-20 更新於 2026-06-18 分類於 資安技術

⚠️ 資安警示：即時通訊視訊監控工具的傳播模式與自保之道

⚠️ 近年來網路上流傳多款號稱可「強制開啟對方視訊」的工具，魚目混珠的情況嚴重，多數夾帶木馬與後門程式。本文明確不提供任何下載連結或軟體名稱，純屬資安意識分享。

⚠️ 到google用key word為"強制視訊"或"視訊強制"找到一堆軟體。
雖然有很多人說這是假的，不過，經過測試確實是存在可以實現強制視訊的版本。
我就不明說哪一版本了，免得遺害大家。
大多為QQ版本，少數為msn版本。

大約有三種版本
1.一種為需數入對方ip，才能觀看對方的視訊。
2.另一種可以直接使用，是隨機抓取的，可直接觀看對方的視訊。
3.還有一種是輸入對方帳號的，才能觀看對方的視訊。

有興趣的，請自行抓取測試。
只測試過一種qq版本，確實可行。(感覺亂可怕的)

P.S.提醒，請在可以馬上還原的電腦上測試，下載點上通常也放了一堆java script來植入木馬，不還原，可能下次就是你被別人看了。



最後，建議大家安裝一個小軟體，就是McAfee SiteAdvisor，可以有效避免一些惡意網站。就算有裝防毒，只要有上網，還是建議安裝一下。個人感覺還不錯用。

傳播模式分析

這類工具的散播通常具有以下特徵，了解這些模式有助於提高警覺：

傳播手法 說明 風險程度
即時通訊好友傳遞 透過 QQ、MSN、微信等軟體傳送所謂「破解版」或「加強版」安裝程式 極高
下載站台暗藏程式碼 所謂的下載頁面內嵌 JavaScript，下載的同時已中木馬 極高
偽裝成系統工具 包裝成「遠端控制伴侶」、「家庭監控軟體」等正規名稱，規避偵測 高
聲稱「真的可用」的行銷文 文章強調「已測試成功」或提供操作截圖，藉此吸引好奇者下載 高
號稱不同版本的話術 以「IP 版、帳號版、隨機版」等區分，製造多版本的假象，誘導反覆下載 高

主要風險

📛 個資外洩

受害者視訊內容遭側錄，可能被用於勒索或公開羞辱。

🦠 惡意程式感染

下載的「工具」本身即為木馬，可能進一步接管系統、成為僵屍網路節點。

🔓 隱私完全暴露

webcam、麥克風、螢幕畫面全數落入攻擊者手中，日常生活一覽無遺。

防護建議

防護項目 說明 操作難度
不使用來路不明的軟體 拒絕透過即時通訊接收的任何「破解工具」，即使是好友傳來的檔案 容易
webcam 電源管理 有實體開關的 webcam 使用完畢後關閉；無開關者可在裝置管理員中停用 容易
作業系統即時更新 保持 OS 與防毒軟體在最新狀態，修補已知漏洞 容易
Mac / Linux 桌面提示燈 部分筆電在 webcam 運作時會亮燈，若燈號異常亮起應立即斷網檢查 容易
定期還原測試環境 如因研究需求須接觸可疑程式，务必在虛擬機或隔離環境中操作，並事後完整還原系統 中等
防火牆嚴格管控 陌生程式要求存取網路時應特別留意；可使用系統防火牆或第三方工具（如 Little Snitch）控管對外連線 中等

推薦工具：McAfee SiteAdvisor

💡 McAfee SiteAdvisor 是一款瀏覽器安全插件，會在搜尋結果頁面即時標示惡意網站，適合一般使用者對抗社交工程攻擊。即便是已有防毒軟體的使用者，也建議安裝作為多一層防護。

功能 說明
搜尋結果標示 在 Google、Bing 結果中以紅/黃/綠標示網站安全性
即時警告 造訪暗藏木馬的下載頁面時自動封鎖並彈出警告
瀏覽器相容 支援 IE、Firefox、Chrome 主流瀏覽器

核心觀念

✅ 這類「強制視訊」工具幾乎不可能同時具備「真的可用」與「完全不帶木馬」兩個條件。即便攻擊者成功製作出不需木馬的版本，受害者下載的「所謂工具」通常也已摻雜惡意程式。

✅ 最有效的自保方式，就是完全不安裝任何來路不明的軟體，不論它是以什麼名義傳來——即便是好朋友「好心分享」，背後也可能已被攻擊者操控帳號。

💜 如有興趣了解 Windows / macOS / Linux 平台的安全防護工具（防毒、惡意程式掃描、網路監控），歡迎參考本系列其他文章。

Simple Install Cacti from source in RHEL / Linux

發表於 2008-10-17 更新於 2026-06-18 分類於 伺服器管理

從原始碼編譯安裝 Cacti（RHEL / Linux）

完整記錄從原始碼編譯安裝 Cacti 網路監控系統的流程，含 MySQL、Apache、PHP、RRDTool、Net-SNMP 等必要元件。

Cacti 需求環境

💡 參考：Cacti 官方需求文件

軟體需求版本
軟體 最低版本
RRDTool 1.0.49 或 1.2.x 以上
MySQL 4.1.x 或 5.x 以上
PHP 4.3.6 以上，強烈建議 5.x
Web Server Apache 或 IIS

下載軟體套件

將所有套件放置於 /usr/local/src 目錄。

軟體 版本 來源
httpd 2.2.9 apache.org
PHP 5.2.6 php.net
MySQL 5.0.67 mysql.com
RRDTool 1.3.4 oetiker.ch
net-snmp 5.4.2 net-snmp.org
libxml2 2.7.2 xmlsoft.org
pango, cairo, freetype, pixman 1.21.1 / 1.6.4 / 2.3.5 / 0.10.0 oetiker.ch/rrdtool/libs

⚠️ 所有套件放置於 /usr/local/src 並切換至此目錄後開始編譯。

cd /usr/local/src

Step 1：安裝 net-snmp-5.4.2

tar zxvf net-snmp-5.4.2.tar.gz
cd net-snmp-5.4.2
./configure --prefix=/usr/local/net-snmp-5.4.2 --with-perl-modules
make
make test   # 請注意是否有 error msg
make install

Step 2：安裝 Apache HTTP Server

tar zxvf httpd-2.2.9.tar.gz
cd httpd-2.2.9
./configure --prefix=/usr/local/httpd-2.2.9
make
make install
ln -s /usr/local/httpd-2.2.9 /usr/local/apache
mkdir /var/www

Step 3：安裝 MySQL 5.0.67

cd /usr/local/src
tar zxvf mysql-5.0.67-linux-i686.tar.gz
mv mysql-5.0.67-linux-i686 /usr/local/
ln -s /usr/local/mysql-5.0.67-linux-i686 /usr/local/mysql
cd /usr/local/mysql
groupadd mysql
useradd -g mysql mysql
chown -R mysql .
chgrp -R mysql .
scripts/mysql_install_db --user=mysql
chown -R root .
chown -R mysql data
bin/mysqld_safe --user=mysql &
cd bin/
./mysqladmin password p@ssw0rd

Step 4：安裝 libxml2（PHP 編譯前置需求）

⚠️ PHP 5.x 編譯需要 libxml2 version 2.6.11 以上，否則會出現：configure: error: libxml2 version 2.6.11 or greater required.

yum install libxml2-devel
tar zxvf libxml2-2.7.2.tar.gz
cd libxml2-2.7.2
./configure --prefix=/usr/local/libxml2-2.7.2
make
make test   # 請注意是否有 error msg
make install

Step 5：安裝 PHP 5.2.6

tar zxvf php-5.2.6.tar.gz
cd php-5.2.6
./configure \
  --prefix=/usr/local/php-5.2.6 \
  --with-mysql \
  --with-apxs2=/usr/local/apache/bin/apxs \
  --with-libxml-dir=/usr/local/libxml2-2.7.2 \
  --with-mysql=/usr/local/mysql \
  --enable-sockets
make
make test
make install
cp php.ini-dist /usr/local/lib/php.ini

修改 httpd.conf 加入 PHP 支援

# vi /usr/local/apache/conf/httpd.conf
LoadModule php5_module libexec/libphp5.so
AddModule mod_php5.c
AddType application/x-httpd-php .php .phtml
AddType application/x-httpd-php-source .phps
AddHandler php5-script .php
AddType text/html .php
DirectoryIndex index.php

測試 Apache + PHP

cd /usr/local/apache/bin/
./apachectl start

建立測試檔案：

# vi /usr/local/apache/htdocs/test.php
<?php
phpinfo();
?>

從瀏覽器存取：http://192.168.70.54/test.php

Step 8：安裝 Cacti 主程式

建立資料庫與使用者

cd /usr/local/src
tar zxvf cacti-0.8.7b.tar.gz
cd /usr/local/mysql/bin
./mysql -u root -p cacti < /usr/local/src/cacti-0.8.7b/cacti.sql

./mysql -u root -p mysql
mysql> GRANT ALL ON cacti.* TO cactiuser@localhost IDENTIFIED BY 'p@ssw0rd';
mysql> flush privileges;
mysql> quit

部署網頁目錄

cd /usr/local/src
mv cacti-0.8.7b /usr/local/apache/htdocs/
ln -s /usr/local/apache/htdocs/cacti-0.8.7b /usr/local/apache/htdocs/cacti

設定 config.php

# vi /usr/local/apache/htdocs/cacti/include/config.php
$database_type     = "mysql";
$database_default = "cacti";
$database_hostname = "localhost";
$database_username = "cactiuser";
$database_password = "p@ssw0rd";

建立 cactiuser 並設定權限

groupadd cactiuser
useradd cactiuser -g cactiuser -s /bin/sh -d /home/users/cactiuser
cd /usr/local/apache/htdocs/cacti
chown -R cactiuser rra/ log/

加入 Crontab

# vi /etc/crontab
*/5 * * * * cactiuser /usr/local/php/bin/php /usr/local/apache/htdocs/cacti/poller.php > /dev/null 2>&1

Step 9：安裝 Cacti Plugin

💡 下載位置：cactiusers.org

所需插件：cacti-plugin-arch.tar.gz、discovery-0.8.4.tar.gz、monitor-0.8.2.tar.gz

Plugin Architecture（必備，優先安裝）

tar zxvf cacti-plugin-arch.tar.gz
cd cacti-plugin-arch
cp cacti-plugin-0.8.7b-PA-v2.1.diff /usr/local/apache/htdocs/cacti/
cd /usr/local/apache/htdocs/cacti
patch -p1 -N <>
cd /usr/local/src/plugin/cacti-plugin-arch/
/usr/local/mysql/bin/mysql -u root -p cacti <>

修改 global.php：

# vi /usr/local/apache/htdocs/cacti/include/global.php
$config['url_path'] = '/';    →   $config['url_path'] = '/cacti/';

Syslog 整合至 Cacti（網路設備適用）

💡 此功能適合用於 Network Device，大量 Host Device 不建議使用。

安裝 Syslog 插件

tar zxvf syslog-0.5.2.tar.gz
mv syslog /usr/local/apache/htdocs/cacti/plugins/
# vi /usr/local/apache/htdocs/cacti/include/global.php
$plugins[] = 'syslog';

建立 syslog 資料庫

mysql -u root -p
mysql> create database syslog;
mysql> GRANT ALL ON syslog.* TO cactiuser@localhost IDENTIFIED BY 'p@ssw0rd';
mysql> flush privileges;

# mysql –u root –p syslog < [syslog sql file]

修改 Syslog 插件設定

# cd /usr/local/apache/htdocs/cacti/plugins/syslog
# vi config.php
$syslogdb_type     = 'mysql';
$syslogdb_default = 'syslog';
$syslogdb_hostname = 'localhost';
$syslogdb_username = 'cactiuser';
$syslogdb_password = 'p@ssw0rd';

syslog-ng 設定：寫入 MySQL

# vi /usr/local/etc/syslog-ng/syslog-ng.conf
### 20080116 David ###

### vlan16-1, vlan16-2 ###
destination d_mysql { pipe("/var/log/mysql.pipe" template("INSERT INTO syslog_incoming (facility, priority, date, time, host, message, seq, status) VALUES ( '$FACILITY', '$PRIORITY', '$YEAR-$MONTH-$DAY', '$HOUR:$MIN:$SEC', '$HOST', '$MSG', '$SEQ', '$STATUS' );\n") template-escape(yes));};
destination f_local5 { file("/var/log/vlan16_switch.log"); };
filter f_local5 { facility(local5); };
log { source(src); filter(f_local5); destination(f_local5); destination(d_mysql); };

### 4503-1, 4503-2 ###
destination f_local6 { file("/var/log/vlan57_switch.log"); };
filter f_local6 { facility(local6); };
log { source(src); filter(f_local6); destination(f_local6); destination(d_mysql); };

建立 Syslog to MySQL 轉換腳本

# vi /root/bin/syslog2mysql.sh
================= syslog2mysql.sh =================
#!/bin/sh

if [ ! -e /var/log/mysql.pipe ]; then
  mkfifo /var/log/mysql.pipe
fi

while [ -e /var/log/mysql.pipe ]; do
  /usr/local/mysql/bin/mysql -u cactiuser --password=p@ssw0rd syslog < /var/log/mysql.pipe >/dev/null
done
================= syslog2mysql.sh =================

加入 Crontab（開機自動啟動 + 每分鐘處理）

# vi /etc/crontab
### cacti syslog ###
@reboot root /root/bin/syslog2mysql.sh
*/1 * * * * root /usr/local/bin/php -q /usr/local/share/cacti2/plugins/syslog/syslog_process.php

syslog_incoming 資料表結構

欄位 型別 說明
facility varchar(10) 設施（local5/local6...）
priority varchar(10) 優先等級
date / time date / time 時間戳記
host varchar(128) 來源主機
message text 訊息內容
seq int(10) unsigned AUTO_INCREMENT 序號（PK）
status tinyint(4) DEFAULT 0 處理狀態
